# ACCESS_GATE_QA

승인번호 기능 운영 전 점검 문서입니다.

## 1) 관리자 접속 테스트
- [ ] 승인번호 입력란에 `777200` 입력
- [ ] `승인 후 시작하기` 클릭
- [ ] 접속 성공 후 `관리자 모드` 표시 확인
- [ ] 관리자 전용 `체험용 승인번호 관리` UI 표시 확인

## 2) 0852 차단 테스트
- [ ] 승인번호 입력란에 `0852` 입력
- [ ] 접속 불가 확인
- [ ] 오류/상태 문구로 유효하지 않은 승인번호 처리 확인

## 3) 체험번호 등록 테스트
- [ ] 관리자 모드에서 체험번호 입력 후 추가
- [ ] 체험번호 목록에 신규 번호 표시 확인
- [ ] 목록의 삭제 버튼 동작 확인

## 4) 체험번호 1회 접속 테스트
- [ ] 등록된 체험번호 입력 후 접속
- [ ] 문구 `2회만 체험 가능합니다.` 확인
- [ ] 모드 표시 `체험 모드 · 남은 횟수 1회` 확인

## 5) 체험번호 2회 접속 테스트
- [ ] 같은 체험번호로 다시 접속
- [ ] 모드 표시 `체험 모드 · 남은 횟수 0회` 확인

## 6) 체험번호 3회 차단 테스트
- [ ] 같은 체험번호를 3회째 입력
- [ ] 문구 `체험 가능 횟수 2회를 모두 사용했습니다.` 확인
- [ ] 접속 차단 확인 (intro/layout 미노출)

## 7) 잠금/로그아웃 테스트
- [ ] 상단 `잠금/로그아웃` 클릭
- [ ] `sessionStorage` 승인 상태 초기화 확인
- [ ] access gate 화면으로 복귀 확인
- [ ] 체험 사용 횟수(localStorage)는 유지되는지 확인

## 8) 새로고침 테스트
- [ ] 승인 후 페이지 새로고침
- [ ] 같은 세션에서는 승인 상태 유지 확인

## 9) 보안 주의
- 현재 방식은 `localStorage/sessionStorage` 기반 테스트용 잠금입니다.
- 상업용 전환 시 Apps Script 또는 서버 인증으로 교체가 필요합니다.
- 브라우저 데이터 삭제 시 체험 횟수/세션 상태가 초기화될 수 있습니다.
